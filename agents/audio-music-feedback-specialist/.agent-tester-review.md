# 🔍 Agent Tester Review: `audio-music-feedback-specialist` v0.1.0 (draft)

**Review date**: 2026-07-05
**Reviewer**: agent-tester (subagent)
**Methodology**: Followed `agent-specialist-review` skill — 4-dimension review: harness completeness, non-goal coverage, duplicate scope, path resolution, quality gates / forbidden actions, gaps in role.md / workflow.md.

## Summary

The agent is **well-constructed** and structurally sound. All 13 harness files are present and valid, all paths resolve, boundary audits pass cleanly, format validation succeeds, and no duplicate scope violations were found. The core issue is a **stale `known_failure_carveout`** — the test it references now passes, making dead documentation that could mislead future agents. Additionally, the **Hermes skill wrapper is thin** (51 lines), missing calibration, handoff destinations, and example tasks that a Hermes-only agent would need. The **quality-gate / forbidden_actions mapping** between harness and manifest is asymmetric. Overall: **promotable with minor fixes**.

## Recommendations

1. **[M1] Remove or update the stale `known_failure_carveout`** in `harness.yaml:63`. Since `make check` currently passes 9/9 with 0 failures, either:
   - Remove the carveout entirely (cleanest), or
   - Annotate it: `"# dormant — market-slot test currently passes (Jul 2026); reactivate if it regresses"`

2. **[M2] Fix the dangling reference** in `source-map.md:158`. Either:
   - Add a `capability_inventory:` section to `harness.yaml` and populate it, or
   - Remove the "See `harness.yaml` capability inventory section" sentence.

3. **[M3] Beef up the Hermes skill wrapper** (`.hermes/skills/audio-music-feedback-specialist.md`). At minimum, add:
   - Calibration section (overreach/underreach — already in entrypoint.md, replicate key points)
   - Handoff destinations list
   - Known pitfalls from `workflow.md` (the 10 pitfalls are essential)
   - Verification gates detail (from `rubric.md` Excellent tier)
   - Example tasks (3-5 from `role.md`)
   - Target: bring from 51 lines to 80-100 lines.

4. **[M4] Align quality_gates and forbidden_actions** between `harness.yaml` and `manifest.yaml`:
   - Add `"AGENTS.md Pitfalls updated for any discovered audio error"` to manifest `quality_gates`
   - Either add harness `release_blockers` for the 4 scope-based forbidden_actions (`modify-non-audio-code`, `generate-images`, `write-copy`, `broad-refactors`) or add manifest `forbidden_actions` for the 4 procedural blockers (`spec-not-updated`, `evidence-missing`, `no-old-to-new-params`, `boundary-audit-not-passed`). Consistent mapping is the goal.

5. **[L1] Update crank-feel-specialist role.md** to qualify "soundfx timing" as trigger timing only, deferring sound parameters to audio specialist. The `source-map.md` distinction is correct; the crank-feel role.md should reflect it.

6. **[L2] (Optional) Expand `instructions.md`** to include explicit handoff destinations and 3-5 example tasks from the harness `role.md`. Currently 58 lines — target 75-80.

7. **[L3] (Optional) Consolidate repeated rules** to a single source of truth. Consider adding a `"canonical_rules:"` section to `harness.yaml` and having `role.md`, `tool-policy.md`, and `rubric.md` reference it rather than restating.
