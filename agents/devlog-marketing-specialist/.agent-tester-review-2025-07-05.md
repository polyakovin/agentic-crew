# 🔍 Agent Tester Review: `devlog-marketing-specialist` v0.1.0 (draft)

**Date:** 2025-07-05
**Reviewer:** agent-tester

## Summary

The devlog-marketing-specialist harness is well-structured with clear scope boundaries, comprehensive quality gates, and good documentation across all 13 standard files. No duplicate scope was found with any other specialist in the pack — the public-facing vs in-game boundary with narrative-noir-copy-specialist is crisp and bidirectional handoff is correctly wired. However, three HIGH-severity issues need resolution before promotion to pilot: a YAML quoting bug that corrupts a quality gate in both harness.yaml and manifest.yaml, a stale known_failure_carveout that references a test failure that no longer exists, and a handoff target (visual-art-asset-production-specialist) that doesn't exist yet with no fallback procedure in the devlog-marketing workflow. All paths resolve, all JSON/YAML is syntactically valid, no trailing whitespace, and all 13 harness files are present.

## Findings Sorted by Severity

### 🔴 HIGH

| # | Finding | File(s) | Detail |
|---|---------|---------|--------|
| H1 | **YAML colon-in-value parses as dict, not string** | `harness.yaml:77`, `manifest.yaml:29` | The quality gate `game referred to as Locksmith (hero: Silas Crane)` contains an unquoted colon. YAML interprets this as `{'game referred to as Locksmith (hero': 'Silas Crane)'}` — a dict, not a string. Automated gate-checking code that iterates over `quality_gates` expecting strings will silently skip or crash on this entry. Fix: wrap in quotes — `"game referred to as Locksmith (hero: Silas Crane)"` or use `>-` folded scalar. |
| H2 | **Stale known_failure_carveout — referenced test now passes** | `harness.yaml:62` | The `boot_probe.known_failure_carveout` reads: `"endless_mode.lua rendering test 'market-slot upgrade should render after dynamic rows' is a pre-existing issue outside this specialist's scope."` However, `make test-all` now returns **9 passed, 0 failed** — all tests including endless_mode pass. An agent loading this carveout will expect a failure that isn't there, and may dismiss real failures under the same carveout logic that no longer applies. Remove or update the carveout. |
| H3 | **Handoff target `visual-art-asset-production-specialist` does not exist, no fallback in workflow** | `role.md:37-38`, `entrypoint.md:70-71`, `workflow.md:44` | The agent's role, entrypoint, and workflow all direct the agent to hand off screenshot/asset needs to Visual Art / Asset Production Specialist, but that specialist directory (`/root/agentic-crew/agents/visual-art-asset-production-specialist/`) does not exist. The `health-snapshot.md` acknowledges this gap but the `workflow.md` provides no fallback procedure — the agent will attempt handoff and hit a dead end. Compare with `narrative-noir-copy-specialist`, which documents a fallback: "if unavailable, flag marketing copy to user directly with note." Devlog-marketing needs the same pattern. |

## Recommendations

1. **[H1] Fix YAML quoting**: Change `game referred to as Locksmith (hero: Silas Crane)` to `"game referred to as Locksmith (hero: Silas Crane)"` in both `harness.yaml` and `manifest.yaml`. The colon inside the value must be quoted to prevent YAML parsing it as a mapping key.

2. **[H2] Update or remove stale known_failure_carveout**: `make test-all` now passes with 9/9 suites, 0 failures. The carveout referencing "endless_mode.lua rendering test" is obsolete. Either remove it or replace it with a general note: "Marketing copy does not modify code — build failures do not block drafting unless the feature being described is the failing one."

3. **[H3] Add fallback for non-existent visual-art-asset-production-specialist**: In `workflow.md`, `entrypoint.md`, and `role.md`, add a fallback procedure: "If Visual Art / Asset Production Specialist is not available, document the screenshot need in the report and flag to the user." Mirror the pattern from `narrative-noir-copy-specialist/role.md` line 216.

4. **[M1] Sync manifest quality_gates with harness**: Add the two missing gates to `manifest.yaml`: `no emoji in formal posts (devlog/itch.io)` and `visual asset needs documented, not improvised`.

5. **[M2] Align manifest forbidden_actions with harness release_blockers**: Add to manifest: `use technical jargon in player copy`, `post format not matching template`, `falsify or omit build evidence`, `skip git log review`. Add to harness: `change_game_logic_or_design`, `write_runtime_code`, `use_corporate_marketing_tone`.

6. **[M3] Update narrative-noir-copy-specialist stale references**: In the 4 files that still say devlog-marketing is "planned, not yet created," update to reflect that it now exists. The bidirectional handoff should be fully active.

7. **[M4] Expand Hermes skill**: Add the missing sections from Codex SKILL.md to `.hermes/skills/devlog-marketing-specialist.md`: full non-goals list, tone section, verification gates, and detailed workflow steps. Aim for parity with the Codex wrapper.

8. **[L1] Fix dangling cross-reference**: Either add a `capability_inventory` key to `harness.yaml` or remove the reference from `source-map.md` line 147.

9. **[L2] Consolidate "what to read" list**: Choose `source-map.md` as the single source of truth for the file-read list. All other files should reference it rather than duplicating.

10. **[L3] Run seed evals before promotion**: The 10 eval cases (DM-001 through DM-010) cover all key scenarios including adversarial cases (DM-006, DM-007). Execute at least the critical-path ones before pilot.
