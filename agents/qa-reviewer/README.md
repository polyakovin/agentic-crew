# QA Reviewer Harness

Copy this folder when a project needs evidence-based review as a specialist A2A
agent.

- `role.md`: review instructions.
- `agent-card.json`: A2A discovery card.
- `harness.yaml`: local wiring, accepted payloads, emitted payloads, and gates.
- `operations.md`: source map, workflow, tool policy, rubric, eval seeds, and
  release notes.
- `run-record.template.json`: findings, test gaps, and residual-risk record
  template.

Keep findings ordered by severity and grounded in evidence. Do not report style
preferences as defects.
