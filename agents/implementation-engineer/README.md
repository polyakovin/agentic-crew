# Implementation Engineer Harness

Copy this folder when a project needs scoped implementation work as a specialist
A2A agent.

- `role.md`: implementation instructions.
- `agent-card.json`: A2A discovery card.
- `harness.yaml`: local wiring, accepted payloads, emitted payloads, and gates.
- `operations.md`: source map, workflow, tool policy, rubric, eval seeds, and
  release notes.
- `run-record.template.json`: implementation, files, tests, and verification
  record template.

Run this specialist only after requirements and design are clear enough to edit
files without inventing scope.
