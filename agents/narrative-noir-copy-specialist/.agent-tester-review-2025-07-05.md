# 🔍 Agent Tester Review: `narrative-noir-copy-specialist` v0.1.0 (draft)

**Date:** 2025-07-05
**Review scope:** Full agent harness audit — A2A harness files, Hermes/Codex wrappers, pack routing, scope overlap analysis, adversarial checks, path resolution.

## Summary

The narrative-noir-copy-specialist harness is **well-structured and thorough** across all 13 standard files. The role boundaries are clearly defined, handoff contracts are explicit, and the eval plan has 10 solid seed cases. However, the agent is currently **non-operational** due to a deadlock: the boot probe requires `make test-all` which has a pre-existing rendering test failure the specialist cannot fix. Additionally, a **path resolution bug** in the Hermes manifest will prevent the agent package from loading correctly. Two medium-severity issues (non-ASCII display string, dangling reference) should be resolved before promotion to `pilot`.

## Recommendations

1. **[H1]** **Fix the deadlock**: Either fix the `endless_mode.lua` rendering test, or add a fallback gate to the boot probe — e.g., `make test-all 2>&1 | grep -v "market-slot"` or document the known failure with a skip-list. Without this, the agent is non-operational.

2. **[H2]** **Fix the manifest path**: Change `manifest.yaml:11` from `../skills/narrative-noir-copy-specialist.md` to `../../skills/narrative-noir-copy-specialist.md`. Verify with `realpath`.

3. **[M1]** **Fix the non-ASCII display string**: Change `"Sèvres Enamel"` to `"Sevres Enamel"` (ASCII) or `"Sevres Enamel"` in `economy_catalogs.lua:129`. The agent should catch this in its first audit.

4. **[M2]** **Add Known Pitfalls to `instructions.md`**: Port the Codex `SKILL.md` pitfalls section (lines 79-85) to the Hermes `instructions.md`. Specifically: Playdate Unicode limitation, loreLine opaque-box fit, HUD overflow risk, voice drift warning, and marketing-tone leak warning.

5. **[M3]** **Populate or remove the dangling reference**: Either add a `capability_inventory` section to `harness.yaml` mapping each capability to `use/defer/reject`, or remove the reference from `source-map.md:137-139`.

6. **[M4]** **Add fallback for missing handoff target**: In `role.md` and `entrypoint.md`, note that when `devlog-marketing-specialist` is not yet available, marketing copy should be flagged for the user directly with a note that the specialist is planned but not created.

7. **[L1]** **Scope ASCII boundary audit to exclude comments**: Update the workflow's `rg "[^\x00-\x7F]"` command to pipe through `grep -v "^[^:]*:[0-9]*:--"` to filter out Lua comment lines. Or document Cyrillic comments as known false positives.
