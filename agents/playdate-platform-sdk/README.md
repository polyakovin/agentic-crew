# Playdate Platform / SDK Harness

Copy this folder when a Playdate project needs SDK, platform, simulator/device,
input, storage, graphics, audio, or performance review as a specialist A2A
agent.

- `entrypoint.md`: startup and handoff contract.
- `role.md`: compact role card.
- `agent-card.json`: A2A discovery card.
- `harness.yaml`: local wiring, accepted payloads, emitted payloads, gates, and
  release blockers.
- `source-map.md`: source hierarchy, truth status, freshness, taint boundary.
- `workflow.md`: boot probe, surface classifier, automation packs.
- `tool-policy.md`: allowed tools, approvals, egress, missing-tool behavior.
- `rubric.md`: self-review and challenge checklist.
- `eval-plan.md`: seed eval families and promotion thresholds.
- `run-record.template.json`: structured record for R2/R3/R4 work.
- `release-rollback.md`: package release blockers and rollback steps.
- `health-snapshot.md`: maturity and promotion gaps.

Keep `What To Read` narrow: Playdate docs, project-specific pitfalls, and only
the affected source files for the current task.
