# Product Manager Harness

Copy this folder when a project needs product requirements and acceptance
criteria as a specialist A2A agent.

- `role.md`: requirements instructions.
- `agent-card.json`: A2A discovery card.
- `harness.yaml`: local wiring, accepted payloads, emitted payloads, and gates.
- `operations.md`: source map, workflow, tool policy, rubric, eval seeds, and
  release notes.
- `run-record.template.json`: requirements and acceptance-criteria record
  template.

Keep `What To Read` limited to product specs, user-facing docs, and prior
decisions relevant to the task.
